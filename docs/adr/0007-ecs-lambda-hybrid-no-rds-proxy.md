# ADR-0007: バックエンドはECS(API本体)とLambda(非同期処理)のハイブリッド構成とし、RDS Proxyは採用しない

## ステータス

承認済み

## コンテキスト

バックエンドのコンピューティング基盤としてECS FargateとLambdaのどちらかに一本化することを検討していた。判断材料として、個人利用・低トラフィックという前提でのコスト試算を行った(us-east-1想定)。

- Lambda: 個人利用の低トラフィックであれば実行コストはほぼ無料枠内に収まる
- ECS Fargate + ALB: 常時起動のFargateタスク(0.25vCPU/0.5GB)で約$9/月、ALBの固定費が$16〜25/月かかり、合計で約$25〜34/月
- RDS Proxy: 実際の使用量に関わらず、2vCPU分の最低料金として月額$21.90が固定でかかる(Aurora Serverless v2の場合はACU基準)

この試算により、「Lambdaなら圧倒的に安い」という前提はRDS Proxyを組み合わせた場合に大きく崩れることが分かった(Lambda + RDS Proxyで約$22/月 vs ECS + ALBで約$25〜34/月 と、差が数ドル〜10ドル程度に縮小する)。

一方で、機能ごとの性質を見ると以下のように分かれる。

- **家計簿API本体**(収支記録・カテゴリ・予算のCRUD): リクエストのたびに呼ばれる同期処理。安定したDBコネクションプールを持つ常駐プロセスと相性が良い
- **レシートOCR登録**: S3への画像アップロードをトリガーとする非同期処理。呼び出し頻度は低く、イベント駆動の設計が自然
- **月次レポート生成**: 月1回のバッチ処理。スケジュール実行との相性が良い

どちらか一方に無理に統一するより、処理特性に応じて使い分ける方が合理的だと判断した。

## 決定

1. **家計簿API本体はECS Fargateで実装する**。ALB経由でリクエストを受け、RDSへは常駐プロセスのコネクションプールで接続する
2. **レシートOCR登録と月次レポート生成はLambdaで実装する**
   - レシートOCR登録の実装詳細はADR-0005を参照
   - 月次レポート生成の実装詳細はADR-0006を参照
3. **RDS Proxyは採用しない**。理由は以下の2点
   - ECS側は常駐プロセスのコネクションプールを使うため、接続数が跳ねる心配がない
   - Lambda側(OCR登録・月次レポート)もイベント駆動かつ個人利用のため、呼び出し頻度・同時実行数がともに低く、コネクション枯渇のリスクが実質的にない

## 検討した代替案

| 案 | メリット | デメリット | 不採用の理由 |
|---|---|---|---|
| ECS + Lambda ハイブリッド、RDS Proxyなし(採用) | 処理特性に応じた合理的な使い分け、コンテナ/サーバーレス両方の実装力を見せられる、コストも最小限 | 構成要素が増え、インフラのコード量(Terraform)がやや増える | — |
| 全てECS Fargateに統一 | 構成がシンプル、コネクション管理を1箇所で完結できる | イベント駆動の処理(OCR登録、月次バッチ)をECS上で動かすには別途スケジューラ実装が必要になり、Lambdaより実装コストが高い | 処理特性に合わないため不採用 |
| 全てLambdaに統一 | コストは最小、サーバーレスの実装力を見せられる | 常時稼働のAPIをLambdaで実装するとコールドスタートの影響を受けやすく、ローカル開発(docker-compose)との一貫性も崩れる | 同期APIの特性に合わないため不採用 |
| ECS + Lambda ハイブリッド、RDS Proxyあり | 接続数が跳ねた場合の保険になる | 個人利用では発生しない問題に対して月$21.90の固定費を払うことになり、コスト効率が悪い | リスクに対してコストが見合わないため不採用 |

## 結果・影響

- インフラ構成要素(ECS、Lambda×2、S3、EventBridge、Textract)が増えるため、Terraformのモジュール分割(`infra/modules/`)を丁寧に行う必要がある
- 「なぜコンピューティング基盤を1つに統一しなかったか」を面接で問われた場合、本ADRの試算とコンテキストがそのまま回答になる
- 将来、家族での共有機能などでアクセス数が増えた場合は、ECS側のコネクション数を再計測し、RDS Proxy導入を再検討する(その際は新たにADRを起票する)
- 本ADRはコンピューティング基盤(ECS/Lambda)の使い分けとRDS Proxy不採用の根拠を整理したものであり、サブスクスケジューリング(ADR-0003)・レシートOCR(ADR-0005)・月次レポート(ADR-0006)の個別実装判断はそれぞれのADRを参照する
