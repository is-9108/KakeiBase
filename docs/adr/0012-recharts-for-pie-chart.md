# ADR-0012: カテゴリ別円グラフに Recharts を採用する

## ステータス

承認済み

## コンテキスト

ダッシュボード画面（Issue #11）でカテゴリ別支出の割合を視覚化する円グラフが必要になった。
React/TypeScript 環境で利用できるチャートライブラリを検討した。

## 決定

Recharts を採用し、`PieChart` / `Pie` / `Cell` コンポーネントで円グラフを実装する。

## 検討した代替案

| 案 | メリット | デメリット | 不採用の理由 |
|---|---|---|---|
| Recharts | React ネイティブ設計、TypeScript 型定義内蔵、PieChart が標準提供、ドキュメント充実 | バンドルサイズ増加 (~200 KB gzip 前後) | — |
| SVG 自作 | 依存ゼロ | アニメーション・ツールチップ等の実装コストが高い | ポートフォリオとして実装工数に見合わない |
| Chart.js | 実績豊富 | React 向けに `react-chartjs-2` ラッパーが別途必要、依存が2層になる | 採用理由の優位性が薄い |

## 結果・影響

- `recharts` が `dependencies` に追加される
- バンドルサイズは増加するが、ポートフォリオ用途では許容範囲
- 将来的に他のグラフ（折れ線グラフ等）が必要になった場合も同一ライブラリで対応できる
- Recharts のメジャーバージョンアップ時に API 変更が伴う可能性がある
