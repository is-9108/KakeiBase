# 開発ログ

実装の裏側にある試行錯誤・詰まった点・学びを週次で記録する。ADRが「決定の記録」なのに対し、こちらは「過程の記録」。面接で「開発中に困ったことは？」と聞かれた時にそのまま使える。

## 書き方

- 週の終わりに5〜10分で書く。完璧を目指さない
- やったこと / 詰まったこと / 学び / 次週やること、の4項目で十分
- 該当するPRやADR、Discussionへのリンクを入れる
