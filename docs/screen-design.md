## 画面遷移図

```
[未認証]
    │
    └── /login (ログイン画面)
              │
              │ ログイン成功
              ▼
[認証済み]
    │
    ├── / (ダッシュボード) ◄────────── グローバルナビ
    │         │
    │         └── 収支一覧リンク
    │
    ├── /transactions (収支一覧)
    │         │
    │         ├── 新規登録モーダル (開く/閉じる)
    │         └── 編集モーダル (開く/閉じる)
    │
    ├── /categories (カテゴリ管理)
    │         │
    │         └── カテゴリ追加モーダル
    │
    └── /subscriptions (サブスク管理)
              │
              └── サブスク追加/編集モーダル

[認証済み → 未認証]

    アクセストークン期限切れ → /api/auth/refresh → 失敗時 /login にリダイレクト
```

## 画面詳細

### 1. ログイン画面 (`/login`)

**目的:** メールアドレスとパスワードで認証する。

**ワイヤーフレーム:**

```
┌─────────────────────────────────────┐
│                                     │
│           KakeiBase                 │
│                                     │
│  ┌─────────────────────────────┐   │
│  │  メールアドレス               │   │
│  │  [________________________] │   │
│  │                             │   │
│  │  パスワード                   │   │
│  │  [________________________] │   │
│  │                             │   │
│  │  [      ログイン      ]      │   │
│  └─────────────────────────────┘   │
│                                     │
│  ※ ログインに失敗しました (エラー時)  │
│                                     │
└─────────────────────────────────────┘
```

**インタラクション:**
- ログインボタン押下 → `POST /api/auth/login`
- 成功: ダッシュボード (`/`) へリダイレクト
- 失敗: エラーメッセージ表示（認証情報の詳細は開示しない）

## 共通 UI 仕様

### グローバルナビゲーション

全認証済み画面に表示するヘッダー。

```
┌──────────────────────────────────────────────────────────────┐
│  KakeiBase  [ダッシュボード] [収支] [カテゴリ] [サブスク]  [ログアウト] │
└──────────────────────────────────────────────────────────────┘
```

### ローディング・エラー状態

- API リクエスト中: スピナー表示
- API エラー時: トースト通知でエラーメッセージを表示
- 401 Unauthorized: `/api/auth/refresh` を試み、失敗したら `/login` にリダイレクト

### レスポンシブ対応

- ブレークポイント: モバイル (< 768px) / デスクトップ (≥ 768px)
- モバイル時: ナビゲーションはハンバーガーメニューに変更
