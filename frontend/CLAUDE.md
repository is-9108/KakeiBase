# フロントエンド コーディング規約

## コーディング規約(React/TypeScript)

- 関数コンポーネント + Hooksのみ(クラスコンポーネント禁止)
- APIアクセスは `src/api/` に集約し、コンポーネントから直接fetchしない
- 状態管理はまずReact標準(useState/useContext)で足りるかを検討し、安易にライブラリを増やさない
- Tailwindなどのユーティリティクラスを使う場合はコンポーネント内で完結させる

## コメント規約

コードの意図が自明でない場合は、以下の観点でJSDocコメントを付ける。

- **プロパティ / 型のフィールド**: 何を格納するためのフィールドなのかを明記する
  ```ts
  type Expense = {
    /** 支出が発生した日付 (ISO 8601形式) */
    date: string;
    /** 支出金額 (円, 正の整数) */
    amount: number;
  };
  ```

- **関数 / カスタムHook**: 何をする関数なのかを明記する
  ```ts
  /**
   * 指定月の支出一覧を取得するカスタムHook
   */
  export function useExpenses(month: string) { ... }
  ```

- **引数**: 何を渡すパラメーターなのかを `@param` タグで明記する
  ```ts
  /**
   * @param month 集計対象の年月 (例: "2026-07")
   */
  ```

- **戻り値**: 何を返すのかを `@returns` タグで明記する
  ```ts
  /**
   * @returns 予算残高(円)。予算未設定の場合は undefined
   */
  ```

> `id`, `name` など意味が自明なフィールドや、Props型の単純なプロパティは省略してよい。
> コンポーネントのProps型には、自明でないプロパティにコメントを付ける。
